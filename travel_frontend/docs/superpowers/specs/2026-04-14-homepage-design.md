# 比赛版首页设计说明

## 目标

首页定位为比赛展示用的高级质感演示页，而不是传统企业后台或普通产品首页。页面需要同时完成三件事：

1. 在首屏建立强烈的作品气质和旅行氛围。
2. 清晰传达产品核心能力是 AI 旅行规划。
3. 将用户自然引导到生成流程、案例区和后续工作台。

## 设计结论

首页采用以下整体结构：

- 左上固定品牌 Logo / 品牌名。
- 左侧固定极简导航，采用图标加简短文字。
- 首屏使用多目的地可切换的大图氛围 Hero。
- 中央保留大标题、副标题和超大输入框。
- 首屏下方继续向下滚动展示能力卡片和精选案例。

这是一个“氛围型展示页 + AI 生成入口 + 轻导航框架”的组合方案。

## 页面结构

### 1. 顶部品牌区

- 页面左上角仅保留 Logo 和品牌名。
- 不放登录、注册、用户菜单等常规站点操作。
- 顶部区域服务于品牌识别，不承担导航主功能。

### 2. 左侧固定导航

左侧导航为全程固定悬浮式导航，和左上品牌区在视觉上保持同一列起点。

导航规则：

- 使用窄栏布局，保持弱存在感。
- 每项采用图标 + 2 到 3 字文字。
- 默认低对比显示，hover 和当前态增强。
- 不做厚重纯色侧边栏。
- 不放复杂状态区、头像区和多层级菜单。

建议导航项：

- 首页
- 灵感
- 生成
- 案例
- 行程

导航行为：

- 首页：滚动回 Hero 区。
- 灵感：滚动到目的地 chips / prompt 区。
- 生成：进入规划页或工作台。
- 案例：滚动到精选案例区。
- 行程：跳转到我的行程页。

### 3. Hero 首屏

Hero 是页面视觉核心，采用全屏或接近全屏的沉浸式大图作为背景。

背景规则：

- 支持 3 到 5 个目的地主题切换。
- 主题示例可为东京、巴黎、冰岛、海岛、雪山湖泊。
- 图片需要统一色调和遮罩处理，避免素材拼盘感。
- 切换方式以缓慢淡变为主，不使用生硬轮播。

Hero 内容：

- 主标题：一句简洁、适合投屏的核心价值表达。
- 副标题：补充说明 AI 会生成专属行程、路线和建议。
- 超大输入框：作为页面最核心的交互入口。

推荐表达方向：

- 主标题聚焦“生成旅程”而不是堆砌功能名词。
- 文案保持克制和高级感，不使用冗长宣传语。

### 4. 超大输入框

输入框是首页的产品中心，而不只是一个视觉装饰。

输入区目标：

- 让用户一眼理解可以从这里开始生成。
- 保持高级感，同时具备明确的可交互性。

输入区组成：

- 主要提示文案。
- 清晰的主操作按钮。
- 轻量模式或旅行意图标签。
- 目的地或旅行风格 chips。

推荐交互：

- 点击目的地 chip 时同步切换背景图和主题色。
- 点击 prompt 可自动填入输入框。
- 点击主按钮进入规划流程。

### 5. 首屏下方能力卡片

Hero 下方不做大段功能介绍，而是保留 3 到 4 张轻量能力卡片。

建议内容：

- 智能生成行程
- 地图联动路线
- 预算与交通建议
- 灵感案例推荐

这些卡片只承担“补充可信度和完整性”的作用，不与 Hero 抢戏。

### 6. 精选案例区

首页首屏之后继续向下滚动进入案例区，案例区是比赛答辩的重要支撑内容。

案例区规则：

- 使用高质量封面图展示不同旅行主题。
- 每张案例卡展示目的地、行程天数、风格标签。
- 点击后进入案例详情或工作台预览。
- 保持内容数量克制，避免做成信息流或瀑布流社区。

建议案例方向：

- 东京 3 天城市漫游
- 巴黎 5 天浪漫假期
- 冰岛 7 天环岛自驾
- 海岛度假 4 天轻松疗愈

### 7. 页面尾部 CTA

页面尾部保留一个明确的再次转化入口。

示例方向：

- 立即生成我的旅行计划
- 从一句话开始规划旅程

该区域用于把浏览案例后的用户引回主路径。

## 视觉语言

整体视觉采用“高端旅行海报感 + 轻玻璃感 + 克制产品界面”的路线。

视觉原则：

- 大图氛围优先。
- 中心构图优先。
- 留白充足。
- 动效克制。
- 颜色柔和统一。

不采用以下方向：

- 传统后台式布局
- 纯内容社区首页
- 过重的赛博或炫技风格
- 过度玻璃拟态导致可读性下降

### 色彩与质感

- 基础色以暖白、雾灰、浅金、海盐蓝为主。
- 不同目的地允许做轻微主题偏色，但必须维持统一品牌感。
- 输入框和卡片可使用轻度玻璃感和微描边。
- 文本和按钮必须保持足够对比度，不能因氛围而牺牲阅读性。

### 动效

仅保留三类主要动效：

- 背景图淡入淡出切换
- Hero 文案与输入区的轻微上浮显现
- chips / 导航项 / 卡片的柔和反馈

避免使用夸张位移、频繁缩放、强烈粒子或炫目的过场动画。

## 信息层级

页面优先级必须保持明确：

1. 背景大图氛围
2. 主标题
3. 超大输入框
4. 目的地 chips / prompt
5. 左侧导航
6. 能力卡片和案例

左侧导航的职责是导览，不应成为视觉主角。

## 响应式策略

桌面端以完整体验为主。

移动端处理原则：

- 左侧导航收缩为底部浮层或顶部展开入口。
- Hero 保留大图和标题，但缩短纵向高度。
- 输入框改为单列堆叠。
- chips 支持横向滚动。
- 案例区改为单列卡片。

移动端应保留设计语言，但不强求桌面端相同的沉浸感。

## 实现边界

首页实现优先采用稳定、可控的本地配置，不依赖复杂远程动态能力。

推荐实现方式：

- `HomePage.vue` 作为页面容器和编排层。
- Hero 区抽离为独立组件，例如 `components/home/HomeHero.vue`。
- 左侧导航抽离为独立组件，例如 `components/home/HomeSideNav.vue`。
- 案例区抽离为独立组件，例如 `components/home/HomeCasesSection.vue`。
- 目的地切换与主题配置抽离到 composable，例如 `composables/useHomeDestinations.ts`。

目的地配置建议统一数据化：

- id
- name
- title
- subtitle
- background image
- theme color
- prompt list
- case preview

## 与现有项目的衔接

首页需要与现有规划页、工作台、行程页形成统一产品链路。

建议关系：

- 首页负责展示和引导。
- 规划页 / 工作台负责深入操作。
- 行程页负责结果留存和回看。

首页视觉应比内页更强，但基础色彩、圆角、卡片语言和交互细节应可延续到后续页面，避免像两个独立产品。

## 成功标准

当首页实现完成后，应达到以下标准：

1. 首屏在静态展示下具备明显比赛级视觉吸引力。
2. 用户在 3 秒内能理解这是 AI 旅行规划产品。
3. 用户能在首屏直接开始输入或切换旅行灵感。
4. 向下滚动后能够看到案例支撑，而不是只停留在一张海报。
5. 首页和后续工作台之间不存在明显风格断裂。
